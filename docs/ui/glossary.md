# UI 用語集

プロダクト内で利用する主要用語と表記ルールをまとめます。デザイナー・エンジニア・ライターはこの一覧を参照し、UI の文言揺れを防止してください。

| 用語 | 表記 | 意味 / 文脈 | 使用上のルール |
| --- | --- | --- | --- |
| プロダクト名 | 性癖ラボ | アプリケーション全体のサービス名。ロゴやヒーロー文言にも使用。 | 常に全角。「性癖ラボへようこそ」など語尾に句点は付けない。 |
| スワイプ | スワイプ | `/swipe` でのカード操作。LIKE/NOPE の動作全般を指す。 | 名詞としては「スワイプ体験」。動詞化しない（例: ×「スワイプする」→○「カードをスワイプ」）。 |
| LIKE / NOPE | LIKE / NOPE | 好みを示す2種類のアクションラベル。 | 英大文字表記を維持。説明文では「LIKE（お気に入り）」「NOPE（スキップ）」など補足可。 |
| 決定枚数 | パーソナライズ枚数 | パーソナライズ目標の進捗表示に利用。 | 「あと◯枚でパーソナライズ完了」のように残数とセットで使用。 |
| AI検索 | AI検索 | `/search` ページ全体で表示される機能名称。 | 常に大文字 `AI` + 漢字。「AI 検索」とスペースは挿入しない。 |
| みんなが見ているトレンド | みんなが見ているトレンド | `/search` のトレンド棚。非ログインでも表示するセクション。 | セクションタイトルは固定。説明文では「トレンド棚」と略してもよい。 |
| 新着 | 新着 | `/search` の最新作品セクション。 | セクション見出しは 4 文字固定。補足が必要な場合は「新着（公開順）」と括弧書き。 |
| 気になるリスト | 気になるリスト | `/lists` で LIKE 作品を振り返る機能。 | 文章中では「気になるリストで確認」のように名詞扱い。英語表記は使用しない。 |
| 嗜好レポート | 嗜好レポート | `/insights` で提供する可視化ページ。 | タイトルは「あなたの性癖レポート」など状況に応じて変えても良いが、「嗜好レポート」という単語は原則保持。 |
| アカウント管理 | アカウント設定 | サイドバー／ヘッダーから開くプロフィール・サブスク設定 UI。 | 「アカウント設定」で統一し、「プロフィール設定」「ユーザー設定」とは記載しない。 |
| 通知トースト | トースト通知 | `react-hot-toast` によるフィードバック。 | 通知本文は 20 文字程度を目安に簡潔に。冒頭は動詞ではなく「保存しました」のように結果の述語で始める。 |
| ログインモーダル | ログイン / 新規登録 | 認証モーダル全般。 | 文言としては「ログイン / 新規登録」をフルで表示する。案内文で「サインイン」は使用しない。 |

## 運用ルール

1. 新しいページや機能を追加する際は、このファイルに用語を追加してから UI 実装に着手する。
2. 複数の候補語がある場合は、ユーザーの認知負荷が低い簡潔な表現を優先し、備考欄に説明ルールを残す。
3. 多言語展開時はここをソース・オブ・トゥルースとして翻訳メモリに連携する。
