# 性癖ラボ 初期戦略仮説と評価（キュレーションプラットフォーム）

作成日: 2026-06-15

## 1. 戦略仮説（要約）

- 性癖ラボは「AVを見る人向けサイト」ではなく、まず「**AVを紹介する人向けプラットフォーム**」を目指す。
- 初期価値はレコメンドでも検索でもなく **キュレーション**（例: 黒髪清楚100選、元アイドル系、彼女感重視、添い寝系まとめ等）。
- 核心仮説: 性癖ラボは「**Xのフロー型コンテンツをストック型資産へ変換する場所**」になる。
- 初期ターゲット: 一般視聴者ではなく **AV紹介アカウント / アフィリエイター / 女優紹介アカウント / AVオタク**。
- 初期KPI: PVではなく供給側（リスト作成者数・アクティブキュレーター数・継続率・作成リスト数）。

### コールドスタート問題（仮説の前提）
- Netflix型レコメンドはユーザー数・行動履歴・購買履歴が大量に必要 → 初期には機能しない。
- 「AV検索エンジン」は実態として正しいが訴求が弱い（ユーザーは「検索したい」のではなく「見たい作品/女優を見つけたい」）。

## 2. 現状のサイト構造（コードから確認）

キュレーション中核はすでに実装済み。ピボットコストは低い。

- 実装済み: `frontend/src/app/list/[token]/`（リスト表示・`ListEditMode`・`ListStats`・OG画像）、`my/lists/`、`u/[username]/`（公開プロフィール・編集モード）、`api/affiliate/`。
- 視聴者向け消費導線: `quiz`（性癖診断）・`swipe`・`explore`・`videos`・`performers`・`tags`・`browse`・`grid`。

→ **「キュレーション機能はあるが、キュレーター獲得装置になっていない」**状態。トップ導線の重心が視聴者向けに寄っている。

## 3. 評価（7論点）

1. **成立しうるか** … 条件付きYES。最大の壁は需要側でなく **供給側の継続（2本目を作るか）**。リストが作成後に勝手に育ち、閲覧/保存/流入が作者に可視化フィードバックされることが生命線。
2. **「フロー→ストック変換」の価値提案の強さ** … 方向は正しいが訴求は弱い。「資産化」は運営の論理で、キュレーター本人の言葉ではない（彼らが欲しいのはフォロワー・反応・収益・承認）。note/WordPress/Lit.Link 等の代替もあるため、**横断検索・タグ・類似女優グラフという発見導線（ディストリビューション）に乗る差分**を価値の中心に据える。
3. **ターゲットを視聴者でなく紹介アカウントに置く妥当性** … 妥当（two-sided market の正しい順序）。ただし「フォロワーを他所へ送りたくない」力学に注意。**作者クレジット＋Xへの送客強化**と**アフィリIDのパススルー（収益を奪わない）**が必須。
4. **参加インセンティブ設計（優先度順）** … ①アフィリ収益パススルー（最強）②配信ブースト（作れば人が来る）③作者の可視化（`u/[username]` をポートフォリオ化、実績バッジ）④作成コスト極小化（Xから半自動取り込み）⑤承認・ランキング。本命は①金銭と②流入、③〜⑤は補助。
5. **構造の整合性** … 部分整合。中核（リスト/プロフィール/統計/アフィリ）は存在するが、重心が視聴者向け消費に寄る。欠けている可能性: ①Xからのリスト取り込み導線 ②アフィリIDパススルー設定 ③作者実績可視化 ④診断結果/検索からリストへの送客。作り直しでなく**重心移動**で対応可能。
6. **初期KPI** … 北極星候補=**継続キュレーター数**（月内2本以上作成した人数）。先行指標=新規作成者数・2本目転換率・リスト単位の閲覧/保存/アフィリクリック。遅行=PV/UU/CTR/成果。初期にPVを北極星に置かない。
7. **最も危険な箇所（危険度順）** … ①キュレーターは「資産化」を求めていない可能性（本当の動機がフォロワー獲得・収益なら"中間業者"扱いされ最重要層が来ない）②2本目の壁＝供給継続が立たない③コンテンツ陳腐化・法務/プラットフォーム（決済・SEO・X規約）リスク。

## 4. 総括と次の一手

- 論理は健全、コード資産も揃っており **ピボットコストは低い**。
- 最大リスクは戦略でなく「**最も来てほしいキュレーターの動機を読み違える**」こと。
- **次の一手は実装ではなく検証**: Xの紹介アカウント/アフィリエイター 5〜10人へのヒアリングで論点1・3・7-①を確かめてからスケールする。
